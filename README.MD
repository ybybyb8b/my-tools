# 🛠️ 我的工具箱 - 数据修改指南
这里记录了如何修改两个工具的菜单数据。修改后，点击 Commit changes，等待 1-2 分钟，刷新网页即可生效。

## 🥗 1. 晚餐决策助手 ( `dinner/menu_data.json` )
这个文件控制“今天吃什么”的抽签逻辑。

**核心概念：权重 (weight)**
- 什么是权重？ 相当于“抽奖券”的数量。
- 例子：如果 A 的权重是 `1` ，B 的权重是 `5` 。那么 B 被抽到的概率是 A 的 5 倍。
- 怎么改？ 想常吃就改大，不想吃就改小（最小为 1）。

### 参数说明
文件分为三个大部分：

**A. 情景模式 ( `solo` 和 `shared` )**
- `solo` : 自己独享时的设置。
- `shared` : 二人食时的设置。
- 里面的参数：
    - `takeout` : 外卖池。
        - `"item"` : 店名或菜名。
        - `"weight"` : 抽中概率。
    - `cookTypes` : 自煮类型池（如：炒菜、火锅）。
        - `"type"` : 类型名称（注意：这里的名称必须跟下面 pools 里的逻辑对应，不要乱改名字，只改权重）。

**B. 具体菜池 (`pools`)**
这里是具体的食材和菜谱库。
1. 简单列表（纯文本，无权重）：
- `huncai` (荤菜)、`sucai` (素菜)、`yecai` (叶菜)。
- 修改方法：直接在方括号 `[]` 里加名字，用双引号 `""` 包裹，逗号 `,` 分隔。
- *例子*：`["牛肉", "羊肉", "新增的肉"]`
2. 加权列表（有权重）：
- `zhuchao` (主炒菜)、`fuchao` (辅炒菜)、`mencai` (焖菜)、`sushi` (速食)、`fangbiancai` (方便菜)。
- 修改方法：复制一行大括号 `{...}`，修改里面的 `item` 和 `weight`。

# 👨‍🍳 2. 百年神厨点单 (`chef/dishes.json`)
这个文件是多人聚餐生成菜单的“大数据库”。数据是一个长列表，每一行代表一道菜。

**单道菜的格式**

```json{
  "type": "主荤",
  "name": "红烧肉",
  "price": 48.8,
  "mutex": "猪肉",
  "size": "L"
}```


| 参数详解 | 参数名 | 必填 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| **type** | ✅ | **菜品分类**。只能填以下固定的词:`主荤`、`辅荤`、`凉菜`、`素菜`、`汤品`、`下饭菜`、`小吃`。⚠️ 千万别写错字，否则算法抓取不到。| `"主荤"` |
| **name** | ✅ | **菜名**。 | `"水煮鱼"` |
| **price** | ✅ | **价格**。直接写数字，不要加 ¥ 符号 。| `58.8` |
| **mutex** | ❌ | **互斥标签（防止重复）**。 比如两道菜都是土豆，把它们的 mutex 都设为 "土豆"，系统就绝不会同时点这两道菜。 | `"土豆"` |
| **size** | ❌ | **份量大小**。用于智能判断是大桌菜还是小桌菜。"L" = 大份/硬菜,"M" = 中份（不写默认是这个）,"S" = 小份。 | `"L"` |

## ⚠️ 修改 JSON 的 3 个保命法则
JSON 格式非常严格，错一个标点符号整个网页就会白屏。请务必遵守：
1. 逗号法则：
- 每一行数据的末尾都要加逗号 `,`
- 但是！ 列表里的最后一行绝对不能加逗号。
2.符号法则：
- 所有的冒号 `:`、逗号 `,`、双引号 `""` 必须是**英文符号**。
- 严禁使用中文输入法的全角符号。
3. 引用法则：
- 所有的文字（如 `"牛肉"`）必须用双引号包起来。
- 数字（如 `5` 或 `48.8`）不需要引号。
